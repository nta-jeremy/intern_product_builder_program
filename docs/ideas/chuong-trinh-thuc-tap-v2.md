| **I. TIÊU CHÍ CÁ NHÂN — INTERN SUCCESS PROFILE** |

Đây là bộ tiêu chí cốt lõi để đánh giá sự phát triển của intern trong suốt chương trình. YODY không chỉ đánh giá sản phẩm bạn ship — mà đánh giá cách bạn tư duy, học hỏi và làm việc. Đây là nền tảng để convert sang Full Time.

|  |  |  |  |
|:--:|:--:|:--:|:--:|
| **Khía Cạnh** | **Mô Tả** | **Biểu Hiện Tốt** | **Biểu Hiện Chưa Đạt** |
| **Builder Mindset** | Ship 4 sản phẩm thực từ ý tưởng đến tay người dùng. Không sợ lặp lại. Embrace iteration. | Ship đều, có feedback loop, biết pivot khi cần | Chờ 'hoàn hảo' mới ship. Over-engineer. Sợ sai. |
| **AI-First Thinking** | Sử dụng AI tools thành thạo: Claude, Cursor, n8n… làm đòn bẩy, không phải làm thay. | \>70% feature được build có AI assist, viết prompt tốt | Dùng AI như Google search. Copy-paste mù quáng. |
| **Business Thinking** | Trả lời được WHY (tại sao làm?), WHO (ai dùng?), HOW MUCH (đo được gì?) cho mỗi sản phẩm. | Mọi feature đều có lý do rõ ràng. Biết từ chối scope creep. | Build vì 'thấy hay'. Không biết ai cần cái này. |
| **Learning Agility** | Tự học công cụ mới trong 1-2 tuần. Không cần spoon-feed từng bước. Doc-driven learning. | Gặp tool mới → tự research, thử, fail fast, hỏi thông minh | Chờ tutorial cụ thể. Hỏi trước khi thử. |
| **Ownership** | Chủ động next step mà không cần hỏi. Raise blocker sớm. Đề xuất giải pháp, không chỉ nêu vấn đề. | Tự biết cần làm gì tiếp theo. Báo cáo proactive. | Cần được nhắc từng bước. Biến mất khi bị block. |

| **3.1 HỆ THỐNG ATS (APPLICANT TRACKING SYSTEM)** | **ƯU TIÊN ★** |

Hệ thống quản lý ứng viên từ giai đoạn ứng tuyển đến kết quả tuyển dụng. Giúp HR quản lý toàn bộ pipeline: đăng tin, nhận CV, lưu trữ, phân loại và theo dõi progress từng ứng viên theo đúng quy trình YODY.

**Người dùng chính:**

- Phòng Nhân sự (HR) — Quản lý toàn bộ pipeline tuyển dụng

- Hiring Manager — Xem & đánh giá ứng viên theo từng phòng ban

**Feature MVP cần deliver:**

|  |  |  |  |
|:--:|:--:|:--:|:--:|
| **Feature / Deliverable** | **Output cụ thể** | **KPI / Acceptance Criteria** | **Sign-off** |
| **Problem Discovery & User Flow** | Problem Brief doc, User Flow diagram, Wireframe PDF | HR Lead sign-off trước khi code | HR Lead + PO |
| **User Stories & DB Schema** | 10-15 user stories có AC, DB schema diagram, Tech stack proposal | Tech Lead approve architecture | Tech Lead + PO |
| **Auth + Dashboard UI** | Login page, API docs, Dashboard skeleton | Auth hoạt động, UI match wireframe | Tech Lead |
| **CRUD Candidates + CV Upload** | Candidate list, edit form, CV upload, Status pipeline | Applied → Interviewed → Offer/Reject đúng | Tech Lead + QA |
| **Search, Filter, Notifications** | Search/filter UI, batch action, email triggers | Tìm được ứng viên trong \<3 giây | HR Lead + QA |
| **Analytics + Export** | Dashboard report, time-to-hire, CSV export | Báo cáo tự động, export đúng định dạng | HR Lead |
| **UAT + Deploy Production** | Training docs, deployment checklist, go-live | UAT pass rate ≥80%, zero critical bug | HR Director |

Agent AI tự động kiểm tra chất lượng website YODY (yody.vn): lỗi giao diện, broken links, responsive design, load time, SEO basics. Chạy định kỳ và báo cáo lỗi tự động vào Slack/Email.

| **3.3 THIẾT KẾ VÀ THẨM ĐỊNH QUY TRÌNH** | **ƯU TIÊN ★** |

Đào tạo Agent theo template biểu mẫu nội bộ đã có. Agent đọc hiểu cấu trúc template (phiếu điều chỉnh giá bán, phiếu phê duyệt, phiếu nhập kho...) và tự động kiểm tra dữ liệu điền vào có đúng quy định không — thay thế việc kiểm tra thủ công hiện tại.

**Cách tiếp cận:**

- Thu thập 3-5 template biểu mẫu đang dùng thực tế tại YODY

- Định nghĩa rules/criteria cho từng trường dữ liệu trong biểu mẫu

- Train Agent đọc template → so sánh → flag deviation → alert

- Tích hợp vào workflow hiện tại (không yêu cầu thay đổi quy trình)

**Feature MVP:**

|  |  |  |  |
|:--:|:--:|:--:|:--:|
| **Feature / Deliverable** | **Output cụ thể** | **KPI / Acceptance Criteria** | **Sign-off** |
| **Thu thập & chuẩn hóa Template** | 3-5 template biểu mẫu, criteria specification, rule mapping doc | Compliance Lead confirm criteria đúng | Compliance Lead |
| **Build Agent đọc Template \#1** | Agent code cho biểu mẫu \#1, test result trên sample data | Accuracy ≥80% trên 20 sample data | Tech Lead |
| **Build Agent \#2, \#3 + Alert** | Agents cho 3 biểu mẫu, workflow diagram, Slack/Email alert | 3 agents hoạt động, alert đúng hạn | Compliance Lead |
| **Dashboard + Integration + UAT** | Dashboard thẩm định, integration test, user guide | UAT pass, người dùng tự vận hành được | Compliance Director |

| **3.4 AGENT QUẢN LÝ CÔNG VIỆC VÀ BÁO CÁO CÁ NHÂN** | **TRUNG BÌNH ★★** |

Agent AI giúp từng cá nhân (intern, manager) tự tracking công việc của mình, nhắc nhở deadline, tổng hợp báo cáo tuần/tháng on-demand. Tích hợp với tool quản lý dự án hiện có của YODY — không yêu cầu thay đổi workflow.

**Điểm khác biệt so với báo cáo team:**

- Tập trung vào cá nhân, không phải toàn team

- On-demand: mỗi người tự request báo cáo của mình

- Personalized: format báo cáo tùy theo role (intern ≠ manager)

- Proactive: agent nhắc nhở khi task sắp đến deadline

**Feature MVP:**

|  |  |  |  |
|:--:|:--:|:--:|:--:|
| **Feature / Deliverable** | **Output cụ thể** | **KPI / Acceptance Criteria** | **Sign-off** |
| **Study API & Define Metrics** | API docs, metrics list cho từng role, report template design | PO approve metrics & template | PO |
| **Build Agent Pull Data + Metrics** | Agent code, metrics logic, sample text report | Dữ liệu kéo chính xác, metrics đúng | Tech Lead |
| **HTML Report + Charts + Schedule** | HTML template, chart visualization, cron job | Report đẹp, có biểu đồ, schedule chạy đúng | PO |
| **Email + Filter + Dashboard + UAT** | Email delivery, filter by cá nhân/team, dashboard, training | User tự request được, nhận đúng data | HOD + PO |

| **5.1 Quy Trình Thẩm Định Sản Phẩm** |

Cuối chương trình (Tuần 12), intern sẽ trình bày sản phẩm trước Hội đồng thẩm định. Đây là buổi Product Review chính thức — không phải buổi báo cáo nội bộ. Hội đồng đánh giá độc lập dựa trên tiêu chí đã công bố trước.

|  |  |  |
|:--:|:--:|:--:|
| **Thành Phần Hội Đồng** | **Vai Trò** | **Đánh Giá** |
| **Product Owner (PO)** | Đại diện business — đánh giá sản phẩm có giải quyết đúng bài toán không | Product quality, business fit, user adoption potential |
| **Tech Lead** | Đại diện kỹ thuật — đánh giá chất lượng code, architecture, maintainability | Code quality, AI usage, technical documentation |
| **HR Director** | Đại diện nhân sự — đánh giá sự phát triển cá nhân, ownership, communication | Personal growth, ownership, learning agility |
| **HOD / Stakeholder** | Đại diện người dùng cuối — đánh giá trực tiếp từ góc độ người sử dụng sản phẩm | User satisfaction, ease of use, real-world impact |

**Format buổi thẩm định:**

- Demo live sản phẩm (không slide, không mock): 15 phút mỗi sản phẩm

- Q&A từ Hội đồng: 10 phút mỗi sản phẩm

- Hội đồng chấm điểm theo ScoreCard (Phần VI): riêng tư, không công bố ngay

- Kết quả tổng hợp và feedback được gửi trong vòng 3 ngày làm việc

| **VI. SCORECARD CHI TIẾT** |

ScoreCard được Hội đồng thẩm định sử dụng để chấm điểm tổng kết chương trình. Tổng 100 điểm. Phân bổ: Product Delivery (40đ) + Personal Growth (40đ) + Process Quality (20đ).

|                                    |
|:-----------------------------------|
| **6.1 Product Delivery — 40 điểm** |

|  |  |  |  |  |  |
|:--:|:--:|:--:|:--:|:--:|:--:|
| **Tiêu Chí** | **Điểm** | **1 — Chưa Đạt** | **2 — Đạt** | **3 — Tốt** | **4 — Xuất Sắc** |
| **Mức độ hoàn thành sản phẩm** | **15đ** | \<50% feature planned | 50-70% feature, demo được | 70-90%, staging ready | \>90%, production live |
| **Chất lượng sản phẩm (UAT)** | **10đ** | UAT fail rate \>30% | Fail rate 20-30%, P0 fixed | Fail rate \<10%, zero P0 | Fail rate \<5%, production stable |
| **Stakeholder Satisfaction** | **10đ** | Satisfaction \<3/5 | Satisfaction 3/5 | Satisfaction 3.5/5 | Satisfaction ≥4/5 |
| **Deploy & Production Readiness** | **5đ** | Chỉ là prototype, không deploy được | Deploy được staging | Production ready, có docs | Live, user đang dùng, monitored |

|                                   |
|:----------------------------------|
| **6.2 Personal Growth — 40 điểm** |

|  |  |  |  |  |  |
|:--:|:--:|:--:|:--:|:--:|:--:|
| **Tiêu Chí** | **Điểm** | **1 — Chưa Đạt** | **2 — Đạt** | **3 — Tốt** | **4 — Xuất Sắc** |
| **AI-First Execution** | **10đ** | \<50% feature có AI assist | 50-70%, dùng cơ bản | 70-85%, prompt tốt, cộng tác hiệu quả | \>85%, AI như đồng nghiệp thực thụ |
| **Builder Mindset & Ship Frequency** | **10đ** | Ship \>3 tuần/lần, cần thúc đẩy | Ship 2 tuần/lần, theo sprint | Ship hàng tuần, tự chủ động | Ship 2-3 ngày/lần, lead iteration |
| **Business Thinking** | **10đ** | Không giải thích được WHY, WHO | Giải thích được cơ bản, cần gợi ý | Tự articulate WHY + WHO + đo lường | Full business case, biết từ chối scope creep |
| **Ownership & Proactiveness** | **10đ** | Chờ hướng dẫn từng bước | Respond khi được hỏi | Self-directed, raise blocker sớm | Lead, đề xuất giải pháp, coach người khác |

|                                   |
|:----------------------------------|
| **6.3 Process Quality — 20 điểm** |

|  |  |  |  |  |  |
|:--:|:--:|:--:|:--:|:--:|:--:|
| **Tiêu Chí** | **Điểm** | **1 — Chưa Đạt** | **2 — Đạt** | **3 — Tốt** | **4 — Xuất Sắc** |
| **Template & Documentation Completion** | **10đ** | \<50% template hoàn chỉnh | 50-80%, đủ thông tin cơ bản | 80-95%, rõ ràng, người mới đọc hiểu | 100% + thêm insight & context |
| **Communication Quality** | **5đ** | Chỉ report khi bị hỏi | Update hàng tuần theo lịch | Update 2x/tuần, raise blocker sớm | Proactive, async communication rõ ràng |
| **Learning Agility** | **5đ** | Mất \>3 tuần để master tool mới | 2-3 tuần, cần nhiều support | 1-2 tuần, minimal support | \<1 tuần, tự document lại để share |

|                             |
|:----------------------------|
| **6.4 Thang Điểm Tổng Kết** |

|  |  |  |  |
|:--:|:--:|:--:|:--:|
| **Điểm** | **Mức Độ** | **Ý Nghĩa** | **Convert FT?** |
| **85-100** | **EXCEPTIONAL** | Vượt kỳ vọng ở mọi chiều. Có thể coach người khác. | **Strong Yes — Fast track** |
| **70-84** | **STRONG** | Đáp ứng tốt tiêu chí. Một số điểm nổi bật. | **Yes — Với role phù hợp** |
| **55-69** | **MEETS EXPECTATION** | Hoàn thành nhiệm vụ cơ bản. Cần phát triển thêm. | Maybe — Cần discussion |
| **\<55** | **NEEDS IMPROVEMENT** | Chưa đáp ứng kỳ vọng. Cần review lý do. | No — Feedback & exit |
